    1) Resumen

    • Equipo / Autor(es): Karen N√°jera
    • Curso / Asignatura: Elementos Programables / Proyecto de Control

    • Descripci√≥n breve:
      Este proyecto implementa una plataforma inclinable controlada por 3 servomotores (2 para eje X: izquierdo/derecho y 1 para eje Y: arriba/abajo) cuyo objetivo es mantener una pelota ligera en el centro. Se desarrollaron dos m√©todos de visi√≥n, uno basado en detecci√≥n por color para ubicar una pelota y otro basado en detecci√≥n de l√≠neas para controlar la plataforma con la mano. Adem√°s, se integr√≥ un controlador PD que recibe la posici√≥n detectada por la c√°mara y env√≠a comandos al ESP32 mediante Bluetooth. El ESP32 ejecuta el movimiento con rampado, l√≠mites y retorno autom√°tico a posici√≥n neutral.

    2) Objetivos

    • General:
      Implementar y validar un sistema embebido que, combinando visi√≥n por computadora y control PD, mantenga una pelota centrada sobre una plataforma inclinable y tambi√©n permita un segundo modo de control mediante gestos de la mano.

    • Espec√≠ficos:

    • Dise√±ar la plataforma y la distribuci√≥n de servos (2 en X, 1 en Y).
    • Implementar el nodo de visi√≥n que detecte la pelota y calcule acciones PD.
    • Implementar firmware en ESP32 que reciba comandos por Bluetooth y mueva los servos de forma segura (rampa y l√≠mites).
    • Probar y ajustar ganancias PD para lograr compromiso entre rapidez y suavidad.

    3) Alcance y Exclusiones

    • Incluye:
    • Detecci√≥n de pelota por color con OpenCV (Python).
    • C√°lculo de control PD en el nodo de visi√≥n (Python).
    • Comunicaci√≥n Bluetooth entre PC (visi√≥n) y ESP32.

    • Firmware ESP32 que aplica comandos a 3 servos con l√≠mite, rampa y timeout de seguridad.

    • Exclusiones / restricciones:

    • No se implementa realimentaci√≥n de posici√≥n (no hay encoders).
    • No se implementa autenticaci√≥n/TLS en la comunicaci√≥n.
    • El sistema asume condiciones de iluminaci√≥n y contraste adecuadas para la detecci√≥n por color.

    4) Resultados

    Al ejecutar el sistema: la c√°mara detecta la pelota y el nodo de visi√≥n calcula el error en pixeles respecto al centro de la imagen (X, Y). El controlador PD calcula correcciones y env√≠a comandos DX y DY por Bluetooth al ESP32 (o bien puede enviar directamente √°ngulos AX,AY,AZ). El ESP32 aplica esos comandos a los servos con rampado, l√≠mites de √°ngulo y timeout de seguridad; si no recibe comandos vuelve a la posici√≥n neutral.

    Los par√°metros PD (Kp, Kd), el suavizado EMA, MAX_STEP y MAX_ACCEL permiten ajustar el compromiso entre rapidez y suavidad. Para pelotas muy ligeras las ganancias altas producen movimientos bruscos que lanzan la pelota; por eso recomendamos limitar el paso y usar rampa.

    Durante las pruebas del sistema, la c√°mara logr√≥ detectar de manera efectiva la posici√≥n de la pelota en la imagen y generar un error en los ejes X y Y compar√°ndolo con el centro te√≥rico. El nodo de visi√≥n calcul√≥ las acciones de control PD y las envi√≥ al ESP32 como comandos de correcci√≥n o como √°ngulos finales, dependiendo del modo de operaci√≥n. El ESP32 ejecut√≥ estos movimientos de forma estable gracias al rampado, que evit√≥ cambios de posici√≥n bruscos y mantuvo el movimiento continuo sin sacudidas. La plataforma pudo responder correctamente a los desplazamientos de la pelota, inclin√°ndose de manera proporcional y derivativa para regresar la pelota al centro. El segundo modo de control, basado en la mano del usuario, tambi√©n mostr√≥ un funcionamiento adecuado y permiti√≥ mover la plataforma con libertad mientras se visualizaban las l√≠neas gu√≠a en pantalla. El sistema demostr√≥ ser funcional bajo ambos modos de visi√≥n, cumpliendo con los objetivos planteados.

#include "BluetoothSerial.h"

BluetoothSerial SerialBT;

// === Buffer para lectura BT no bloqueante ===
String btBuffer;

// === Pines de los servos ===
#define SERVO_IZQ   25
#define SERVO_DER   15
#define SERVO_VERT  33

// === PWM ===
const uint32_t FREQ_HZ = 50;
const uint8_t  RES_BITS = 12;
const uint16_t DUTY_MIN = 205;   // ~1.0 ms
const uint16_t DUTY_MAX = 410;   // ~2.0 ms

// Convierte grados f√≠sicos 0..180 a duty
uint16_t dutyFromDeg(int deg){
  deg = constrain(deg,0,180);
  return map(deg,0,180,DUTY_MIN,DUTY_MAX);
}

// Convierte de √°ngulo l√≥gico (0..180, centro 90) a f√≠sico (invertido)
int logicalToPhysical(int logicalDeg){
  logicalDeg = constrain(logicalDeg, 0, 180);
  // 0 l√≥gico ‚Üí 180 f√≠sico, 180 l√≥gico ‚Üí 0 f√≠sico
  return 180 - logicalDeg;
}

// Escribe usando grados l√≥gicos
void writeServoLogical(int pin, int logicalDeg){
  int fisico = logicalToPhysical(logicalDeg);
  ledcWrite(pin, dutyFromDeg(fisico));
}

// Configurar servo con √°ngulo l√≥gico inicial
void configServo(int pin, int initialLogical){
  pinMode(pin,OUTPUT);
  ledcAttach(pin,FREQ_HZ,RES_BITS);   // usa el pin como canal
  writeServoLogical(pin,initialLogical);
}

// === Rango y rampa ===
const int LIM_MIN = 0;
const int LIM_MAX = 180;
const int PASO_RAMPA = 45;          // tama√±o de paso en rampa
const uint32_t DT_RAMP_MS = 2;
const uint32_t TIMEOUT_MS = 700;

// Estado en grados L√ìGICOS
int posIzq = 90;
int posDer = 90;
int posVert= 60;

int tgtIzq = 90;
int tgtDer = 90;
int tgtVert= 60;

uint32_t tPrevRamp = 0;
uint32_t tLastCmd  = 0;

// Rampa suave hacia el objetivo
void aplicarRampa(){
  uint32_t now = millis();
  if(now - tPrevRamp < DT_RAMP_MS) return;
  tPrevRamp = now;

  auto go = [&](int actual,int target){
    if(actual < target) return min(actual + PASO_RAMPA, target);
    if(actual > target) return max(actual - PASO_RAMPA, target);
    return actual;
  };

  posIzq = go(posIzq, tgtIzq);
  posDer = go(posDer, tgtDer);
  posVert= go(posVert,tgtVert);

  // Escribimos usando grados L√ìGICOS, se invierten adentro
  writeServoLogical(SERVO_IZQ, posIzq);
  writeServoLogical(SERVO_DER, posDer);
  writeServoLogical(SERVO_VERT,posVert);
}

// Parsea "ANG:izq,der,vert"
bool parseAngulos(const String &msg, int &aIzq, int &aDer, int &aVert){
  if (!msg.startsWith("ANG:")) return false;
  String data = msg.substring(4);  // despu√©s de "ANG:"

  int c1 = data.indexOf(',');
  if (c1 < 0) return false;
  int c2 = data.indexOf(',', c1 + 1);
  if (c2 < 0) return false;

  String sIzq  = data.substring(0, c1);
  String sDer  = data.substring(c1 + 1, c2);
  String sVert = data.substring(c2 + 1);

  sIzq.trim();
  sDer.trim();
  sVert.trim();

  aIzq  = sIzq.toInt();
  aDer  = sDer.toInt();
  aVert = sVert.toInt();

  return true;
}

// ======== SETUP ========
void setup(){
  Serial.begin(115200);
  SerialBT.begin("ESP32-BallPlatform");

  configServo(SERVO_IZQ, posIzq);
  configServo(SERVO_DER, posDer);
  configServo(SERVO_VERT,posVert);

  Serial.println("ESP32 listo");
  tLastCmd = millis();
}

// ======== LOOP ========
void loop(){

  // --- Lectura Bluetooth no bloqueante ---
  while (SerialBT.available()) {
    char c = (char)SerialBT.read();

    if (c == '\n') {
      // Tenemos una l√≠nea completa en btBuffer
      String msg = btBuffer;
      btBuffer = "";        // limpiar para el siguiente mensaje

      msg.trim();
      if (msg.length() > 0) {
        tLastCmd = millis();

        if (msg == "LOST") {
          // PC perdi√≥ mano/objetivo ‚Üí "centro"
          tgtIzq  = 90;
          tgtDer  = 90;
          tgtVert = 60;
          Serial.println("Comando LOST: centro.");

        } else if (msg == "ZERO") {
          // üîπ TODOS los servos a 180 f√≠sicos
          //    ‚áí 0 l√≥gico por el mapeo invertido
          tgtIzq  = 0;
          tgtDer  = 0;
          tgtVert = 0;
          Serial.println("Comando ZERO: servos ‚Üí 180¬∞ fisico");

        } else {
          int aIzq, aDer, aVert;
          if (parseAngulos(msg, aIzq, aDer, aVert)) {
            tgtIzq  = constrain(aIzq,  LIM_MIN, LIM_MAX);
            tgtDer  = constrain(aDer,  LIM_MIN, LIM_MAX);
            tgtVert = constrain(aVert, LIM_MIN, LIM_MAX);
            Serial.printf("ANG -> IZQ:%d DER:%d VERT:%d\n", tgtIzq, tgtDer, tgtVert);
          } else {
            Serial.print("Comando desconocido: ");
            Serial.println(msg);
          }
        }
      }
    } else if (c != '\r') {
      // Acumulamos caracteres, ignorando CR
      btBuffer += c;
    }
  }

  // Si pasa mucho tiempo sin recibir comandos, vuelve al centro
  if(millis() - tLastCmd > TIMEOUT_MS){
    tgtIzq = 90;
    tgtDer = 90;
    tgtVert= 60;
  }

  aplicarRampa();
  delay(1);
}

    dx = x2 - x1
    dy = y2 - y1

// Convierte grados f√≠sicos 0..180 a duty
uint16_t dutyFromDeg(int deg){
  deg = constrain(deg,0,180);
  return map(deg,0,180,DUTY_MIN,DUTY_MAX);

// Convierte de √°ngulo l√≥gico (0..180, centro 90) a f√≠sico (invertido)
int logicalToPhysical(int logicalDeg){
  logicalDeg = constrain(logicalDeg, 0, 180);
  // 0 l√≥gico ‚Üí 180 f√≠sico, 180 l√≥gico ‚Üí 0 f√≠sico
  return 180 - logicalDeg;
}

// Escribe usando grados l√≥gicos

}

  auto go = [&](int actual,int target){
    if(actual < target) return min(actual + PASO_RAMPA, target);
    if(actual > target) return max(actual - PASO_RAMPA, target);
    return actual;
  };

  posIzq = go(posIzq, tgtIzq);
  posDer = go(posDer, tgtDer);
  posVert= go(posVert,tgtVert);

  // Escribimos usando grados L√ìGICOS, se invierten adentro
  writeServoLogical(SERVO_IZQ, posIzq);
  writeServoLogical(SERVO_DER, posDer);
  writeServoLogical(SERVO_VERT,posVert);
}

// Parsea "ANG:izq,der,vert"
bool parseAngulos(const String &msg, int &aIzq, int &aDer, int &aVert){
  if (!msg.startsWith("ANG:")) return false;
  String data = msg.substring(4);  // despu√©s de "ANG:"

  int c1 = data.indexOf(',');
  if (c1 < 0) return false;
  int c2 = data.indexOf(',', c1 + 1);
  if (c2 < 0) return false;

  String sIzq  = data.substring(0, c1);
  String sDer  = data.substring(c1 + 1, c2);
  String sVert = data.substring(c2 + 1);

  sIzq.trim();
  sDer.trim();
  sVert.trim();

  aIzq  = sIzq.toInt();
  aDer  = sDer.toInt();
  aVert = sVert.toInt();

  return true;
}

// ======== SETUP ========
void setup(){
  Serial.begin(115200);
  SerialBT.begin("ESP32-BallPlatform");

  configServo(SERVO_IZQ, posIzq);
  configServo(SERVO_DER, posDer);
  configServo(SERVO_VERT,posVert);

  Serial.println("ESP32 listo");
  tLastCmd = millis();
}

// ======== LOOP ========
void loop(){

  // --- Lectura Bluetooth no bloqueante ---
  while (SerialBT.available()) {
    char c = (char)SerialBT.read();

    if (c == '\n') {
      // Tenemos una l√≠nea completa en btBuffer
      String msg = btBuffer;
      btBuffer = "";        // limpiar para el siguiente mensaje

      msg.trim();
      if (msg.length() > 0) {
        tLastCmd = millis();

        if (msg == "LOST") {
          // PC perdi√≥ mano/objetivo ‚Üí "centro"
          tgtIzq  = 90;
          tgtDer  = 90;
          tgtVert = 60;
          Serial.println("Comando LOST: centro.");

        } else if (msg == "ZERO") {
          // üîπ TODOS los servos a 180 f√≠sicos
          //    ‚áí 0 l√≥gico por el mapeo invertido
          tgtIzq  = 0;
          tgtDer  = 0;
          tgtVert = 0;
          Serial.println("Comando ZERO: servos ‚Üí 180¬∞ fisico");

        } else {
          int aIzq, aDer, aVert;
          if (parseAngulos(msg, aIzq, aDer, aVert)) {
            tgtIzq  = constrain(aIzq,  LIM_MIN, LIM_MAX);
            tgtDer  = constrain(aDer,  LIM_MIN, LIM_MAX);
            tgtVert = constrain(aVert, LIM_MIN, LIM_MAX);
            Serial.printf("ANG -> IZQ:%d DER:%d VERT:%d\n", tgtIzq, tgtDer, tgtVert);
          } else {
            Serial.print("Comando desconocido: ");
            Serial.println(msg);
          }
        }
      }
    } else if (c != '\r') {
      // Acumulamos caracteres, ignorando CR
      btBuffer += c;
    }
  }

  // Si pasa mucho tiempo sin recibir comandos, vuelve al centro
  if(millis() - tLastCmd > TIMEOUT_MS){
    tgtIzq = 90;
    tgtDer = 90;
    tgtVert= 60;
  }

  aplicarRampa();
  delay(1);
}
